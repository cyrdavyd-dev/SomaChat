1. Общие положения

Настоящая Политика конфиденциальности описывает, как мобильное приложение Soma Chat («Приложение») обрабатывает информацию на вашем устройстве. Приложение предназначено для локального ведения записей о здоровье в формате чата (сообщения, показатели, симптомы, назначения и связанные вложения).

2. Какие данные обрабатываются

В зависимости от вводимых вами данных на устройстве могут храниться:

  • Данные профиля: отображаемое имя, дата рождения, рост, вес, пол (в виде выбранного варианта), группа крови, ключ страны/региона, аллергии, заболевания, лекарства, локальный путь к фото профиля при его выборе.
  • Содержимое чата: текст сообщений, время, тип сообщения, прикреплённые файлы (копии в защищённой области данных Приложения).
  • Настройки Приложения: например идентификатор активного потока чата для восстановления сессии.

Обработка выполняется на вашем устройстве. В текущей версии Приложения нет учётных записей пользователя и синхронизации с облаком разработчика.

3. Где хранятся данные

Данные хранятся локально в базе SQLite и в файлах в каталоге данных приложения. При формировании экспорта могут создаваться временные файлы в кэше.

Приложение не передаёт ваши медицинские и чат-данные на серверы оператора для хранения. В текущей сборке не используются средства аналитики или отчётов о сбоях, отправляющие персональные данные разработчику.

4. Передача и третьи лица

Оператор по умолчанию не получает ваши данные о здоровье из Приложения. Данные могут покинуть устройство, если вы сами инициируете действие, например:

  • Экспорт / «Поделиться»: PDF и сопутствующие файлы через системное меню — получатель определяется вами (почта, мессенджер, облако и т.д.).
  • Открытие вложения во внешнем приложении средствами ОС.
  • Резервное копирование устройства: при соответствующих настройках данные Приложения могут попадать в резервную копию (в т.ч. облачную).

5. Сеть и разрешения

В версии для Android в манифесте могут быть объявлены разрешения, связанные с сетью; Приложение не использует их для выгрузки ваших медицинских записей на серверы оператора.

Доступ к фото для аватара и к файлам для вложений запрашивается только через стандартные средства выбора, когда вы добавляете контент.

6. Безопасность и ответственность пользователя

Обеспечьте безопасность устройства (блокировка экрана, доверенные приложения). Оператор не может гарантировать конфиденциальность данных при компрометации устройства или при передаче экспортированных файлов третьим лицам.

7. Дети

Приложение не ориентировано на сбор данных о детях. Если вы как законный представитель вносите данные ребёнка, вы делаете это добровольно и под свою ответственность.

8. Изменения

Политика может обновляться при изменении функциональности Приложения. Актуальная версия публикуется по тому же URL, который указан в магазине приложений.
